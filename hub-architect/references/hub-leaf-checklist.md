# Hub/Leaf 架构搭建清单

逐步检查，确保架构正确。

## 搭建前

- [ ] 盘点所有现有 Skill，列出触发条件
- [ ] 识别触发词冲突（哪些 Skill 会抢同一个用户意图）
- [ ] 按领域聚类（哪些 Skill 属于同一领域）
- [ ] 确认聚类合理（每个聚类 3-7 个 Skill，太多则拆分，太少则保持独立）
- [ ] 记录基线快照：顶层 Skill 总数 + 各 Hub 计划叶子数

## 搭建中

- [ ] 创建 Hub 目录结构：`{hub}/SKILL.md` + `{hub}/modules/` + `{hub}/references/`
- [ ] Hub SKILL.md 只包含路由逻辑，不含执行细节
- [ ] 每个叶子有独立目录：`modules/{name}/LEAF.md`
- [ ] LEAF.md 保留完整触发条件（可独立使用）
- [ ] 路由规则覆盖所有叶子，无遗漏
- [ ] 路由规则互斥，无重叠触发词
- [ ] 最后一条路由规则是"不确定就问"的兜底

## 迁移（从独立 Skill 到 Hub）

- [ ] **备份先行**：每个要迁移的 Skill 先备份到 workspace/backups/
- [ ] **隔离检查**：备份路径不在 ~/.claude/skills/ 根内
- [ ] **替换验证**：复制到 Hub 后验证 leaf 文件非空
- [ ] **映射更新**：Hub SKILL.md 新增对应映射行 + 路由规则
- [ ] **删除原入口**：验证通过后才删除原 SKILL.md
- [ ] **双根复审**：同时检查 .agents 和 .claude 两个根目录

## 验证

- [ ] **证据三角**：映射证据 + 结构证据 + 行为证据全满足
- [ ] **回归测试**：跑完整回归，至少 6 条/Hub
- [ ] **Gate A**：主路由准确率 >= 92%，高风险域 >= 95%
- [ ] **Gate B**：误触发率 <= 5%，漏触发率 <= 3%
- [ ] **Gate D**：备份目录不在 loader 根内
- [ ] 用模糊触发词测试兜底规则是否生效
- [ ] 确认一次只加载一个叶子
- [ ] 从 Hub 拿出单个叶子，确认可独立使用

## 清理

- [ ] 清理空目录
- [ ] 最终计数（应比基线少）
- [ ] 记录备案清单（phase、日期、迁移列表、备份路径、回归报告）
- [ ] 更新安装文档
