# Hub/Leaf 架构搭建清单

逐步检查，确保架构正确。

## 搭建前

- [ ] 盘点所有现有 Skill，列出触发条件
- [ ] 识别触发词冲突（哪些 Skill 会抢同一个用户意图）
- [ ] 按领域聚类（哪些 Skill 属于同一领域）
- [ ] 确认聚类合理（每个聚类 3-7 个 Skill，太多则拆分，太少则保持独立）

## 搭建中

- [ ] 创建 Hub 目录结构：`{hub}/SKILL.md` + `{hub}/modules/` + `{hub}/references/`
- [ ] Hub SKILL.md 只包含路由逻辑，不含执行细节
- [ ] 每个叶子有独立目录：`modules/{name}/LEAF.md`
- [ ] LEAF.md 保留完整触发条件（可独立使用）
- [ ] 路由规则覆盖所有叶子，无遗漏
- [ ] 路由规则互斥，无重叠触发词
- [ ] 最后一条路由规则是"不确定就问"的兜底

## 搭建后

- [ ] 用每个叶子的典型触发词测试路由是否正确
- [ ] 用模糊触发词测试兜底规则是否生效
- [ ] 确认一次只加载一个叶子（没有同时加载多个的情况）
- [ ] 从 Hub 拿出单个叶子，确认可独立使用
- [ ] 更新安装文档，说明 Hub 的存在和使用方式
